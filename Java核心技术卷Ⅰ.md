## 1.2 Java白皮书[^1]的关键术语
### 1.2.1 简单性
简单来说java是c++的语法优化版,所以若有c++基础,java很容易上手
也建议先学习c++
### 1.2.2 面向对象
面向对象将重点放在数据（对象）和对象的接口上，若有c++基础很好理解
### 1.2.3 分布式
java可以通过URL打开和访问网上的对象，网络已经和变得和呼吸一样简单了！！
### 1.2.4 健壮性
debug友好
### 1.2.5 安全性
java防范的攻击
+ 运行时堆栈溢出
+ 破坏自己的进程空间之外的内存
+ 未经授权读写文件
+ 补充一点，java并非底层语言，无法操作内存
### 1.2.6体系结构中立
java虚拟机（JVM）的实现，使java源文件在即时编译产生的字节码可以直接在有java运行环境的机器上直接运行。这依赖在不同的机器的机器上创建有着相同运行环境的JVM，然后解释虚拟机指令而非运行机器指令[^2]，使java成为非常好的跨平台语言
### 1.2.7可移植性
java规范中没有相对性的地方，举个例子，在c++中，不同机器int类型的大小有所不同，而java中总是32bit。
### 1.2.8 解释性
java虚拟机逐行读取字节码并解释为机器码来执行
### 1.2.9 高性能
java并非传统的编译型或解释型语言，编译产生的字节码可以动态的根据特定cpu和某段代码的执行频率进行优化
### 1.2.10 多线程
java高并发
### 1.2.11 动态性
使用jar包可以避免造轮子

# 第三章 Java的基本程序设计结构
## 3.1 一个简单的Java程序
``` java
public class hello_world {
    public static void main(String[] args) {
        System.out.println("Hello world");
    }
}
```
* 文件名必须与公共类(pulic)名相同，即文件名必须为hello_world.java
* 每个java程序必须包含一个main方法,表示程序入口(与c++不同,java中所有的函数都是某个类的方法,包括main函数(方法),因此需要一个公共类作为外壳作为包含main方法的类)
* main传入的参数是操作系统的指令
* 一个java源文件只能有一个公共类

## 3.3 数据类型
### 3.3.1 整形
|类型|存储需求|取值范围|
|-|-|-|
|int|4字节(32bit)|-2^31^ ~ 2^31^-1|
|short|2字节(16bit)|-2^15^ ~ 2^15^-1|
|long|8字节(64bit)|-2^63^ ~ 2^63^-1|
|byte|1字节(8bit)|-2^7^ ~ 2^7^-1|

[^1]:[Java白皮书](https://www.oracle.com/java/technologies/language-environment.html)
[^2]: 这种方法使java的运行速率比原生的程序慢很多,且不可对内存进行操作
